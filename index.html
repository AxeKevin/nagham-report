SQL1 SQL1 · 渠道概览 — 新增设备行为指标
📋 表：nova.dws_hib_usr_device_active_1d 📅 范围：近 14 天（START_14D ~ T1）

按日期 × 渠道 × 平台 × 代理商聚合新增设备的登录率、进房率、3min渗透率、开口率、发布率。

  • 分母统一为 device_cnt（新增设备数）
  • 3min渗透分母为 room_device_cnt（进房设备数）
  • 增量表，可跨天扫描
SELECT b.p_date, b.af_media_source, b.platform, b.partner_name,
       COUNT(DISTINCT b.device_id)                                                    AS device_cnt,
       COUNT(DISTINCT CASE WHEN b.user_id > 0               THEN b.device_id END)    AS login_device_cnt,
       COUNT(DISTINCT CASE WHEN b.is_room_active > 0        THEN b.device_id END)    AS room_device_cnt,
       COUNT(DISTINCT CASE WHEN b.room_stay_duration >= 180000
                                                             THEN b.device_id END)    AS room_3min_cnt,
       COUNT(DISTINCT CASE WHEN b.recording_prepare_task_cnt > 0
                                                             THEN b.device_id END)    AS solo_open_cnt,
       COUNT(DISTINCT CASE WHEN b.new_item_cnt > 0          THEN b.device_id END)    AS solo_publish_cnt
FROM nova.dws_hib_usr_device_active_1d b
WHERE b.p_date BETWEEN '{START_14D}' AND '{T1}'
  AND b.is_new_device = 1
GROUP BY b.p_date, b.af_media_source, b.platform, b.partner_name
ORDER BY b.p_date DESC, device_cnt DESC
SQL2 SQL2 · 次日留存（T-2 口径）
📋 表：nova.dws_hib_usr_device_active_1d 📅 范围：base = T-2（前天），next_day = T-1（昨天）

次留只查 T-2 单天，避免多天 self-JOIN 超时。T-1 的次留由次日 cron 通过 patch_retention 写回 GitHub JSON。

  • T-2 = 前天，T-1 = 昨天（已完整入仓）
  • 当日 JSON 写入时 T-1 次留 = 0，次日 cron patch_retention 补写
  • 单天查询，执行约 30s
WITH base AS (
    SELECT p_date, af_media_source, platform, partner_name, device_id
    FROM nova.dws_hib_usr_device_active_1d
    WHERE p_date = '{T2M}'       -- T-2 新增设备
      AND is_new_device = 1
),
next_day_active AS (
    SELECT DISTINCT device_id
    FROM nova.dws_hib_usr_device_active_1d
    WHERE p_date = '{T1}'        -- T-1 活跃（今日入仓后即可用）
)
SELECT b.p_date, b.af_media_source, b.platform, b.partner_name,
       COUNT(DISTINCT b.device_id)                                              AS device_cnt,
       COUNT(DISTINCT CASE WHEN n.device_id IS NOT NULL THEN b.device_id END)  AS retention_1d_cnt
FROM base b
LEFT JOIN next_day_active n ON n.device_id = b.device_id
GROUP BY b.p_date, b.af_media_source, b.platform, b.partner_name
ORDER BY device_cnt DESC
SQL3 SQL3 · 广告消耗（4渠道 UNION）
📋 表：ks_origin_nova_sync_db.s_hib_{tiktok/facebook/google/snapchat}_advertise_daily_report 📅 范围：近 14 天（全量表，dt 只取最新分区）

合并 TikTok / Facebook / Google / Snapchat 四渠道的每日广告消耗（USD）。

  • 全量表（snapshot），dt 只取最新分区，date_num 才是数据日期
  • TikTok date_num 格式 YYYY-MM-DD（string）
  • Facebook/Google/Snapchat date_num 格式 yyyyMMdd（bigint）
  • Snapchat spend 单位 micro-cents，÷1e6 转 USD
SELECT af_media_source_mapped, p_date, SUM(spend_usd) AS spend_usd FROM (
  -- TikTok: spend 字段单位 USD（string），date_num 格式 YYYY-MM-DD
  SELECT 'tiktokglobal_int' AS af_media_source_mapped,
         date_num AS p_date,
         SUM(CAST(spend AS DOUBLE)) AS spend_usd
  FROM ks_origin_nova_sync_db.s_hib_tiktok_advertise_daily_report
  WHERE dt = '{T1_DASH}'
    AND date_num >= '{START_14D_DASH}' AND date_num <= '{T1_DASH}'
  GROUP BY date_num
  UNION ALL
  -- Facebook: spend USD (string), date_num bigint yyyyMMdd
  SELECT 'Facebook Ads', CAST(date_num AS STRING),
         SUM(CAST(spend AS DOUBLE))
  FROM ks_origin_nova_sync_db.s_hib_facebook_advertise_daily_report
  WHERE dt = '{T1_DASH}'
    AND date_num >= {START_14D} AND date_num <= {T1}
  GROUP BY date_num
  UNION ALL
  -- Google: cost_micros (string, ÷1e6 = USD), date_num bigint yyyyMMdd
  SELECT 'googleadwords_int', CAST(date_num AS STRING),
         SUM(CAST(cost_micros AS DOUBLE) / 1000000.0)
  FROM ks_origin_nova_sync_db.s_hib_google_advertise_daily_report
  WHERE dt = '{T1_DASH}'
    AND date_num >= {START_14D} AND date_num <= {T1}
  GROUP BY date_num
  UNION ALL
  -- Snapchat: spend bigint micro-cents (÷1e6 = USD), date_num bigint yyyyMMdd
  SELECT 'snapchat_int', CAST(date_num AS STRING),
         SUM(spend / 1000000.0)
  FROM ks_origin_nova_sync_db.s_hib_snapchat_creative_daily_report
  WHERE dt = '{T1_DASH}'
    AND date_num >= {START_14D} AND date_num <= {T1}
  GROUP BY date_num
) t
GROUP BY af_media_source_mapped, p_date
ORDER BY p_date DESC, spend_usd DESC

SQL5 SQL5 · 广告 CTR / CPM（素材级）
📋 表：ks_origin_nova_sync_db.s_hib_{tiktok/facebook/google/snapchat}_advertise_daily_report 📅 范围：近 14 天（全量表）

各渠道广告素材级别的 CTR、CPM，用于素材下钻 Tab 的效果对比。

  • 全量表，dt 只取最新分区
  • CTR = clicks / impressions
  • CPM = spend_usd / impressions × 1000
SELECT af_media_source, p_date, campaign_id, campaign_name,
       adset_id, adset_name, ad_id, ad_name,
       spend_usd, impressions, clicks, ctr, cpm_usd FROM (
  SELECT 'tiktokglobal_int' AS af_media_source, date_num AS p_date,
         campaign_id, campaign_name, adgroup_id AS adset_id, adgroup_name AS adset_name,
         ad_id, ad_name, CAST(spend AS DOUBLE) AS spend_usd, impressions, clicks,
         CASE WHEN impressions>0 THEN clicks*1.0/impressions ELSE NULL END AS ctr,
         CASE WHEN impressions>0 THEN CAST(spend AS DOUBLE)/impressions*1000 ELSE NULL END AS cpm_usd
  FROM ks_origin_nova_sync_db.s_hib_tiktok_advertise_daily_report
  WHERE dt='{T1_DASH}' AND date_num>='{START_14D_DASH}' AND date_num<='{T1_DASH}'
  UNION ALL
  -- Facebook / Google / Snapchat 结构相同，date_num 格式为 bigint yyyyMMdd
  ...（同 SQL3 结构，增加 ad_id/ad_name/impressions/clicks 字段）
) t
ORDER BY p_date DESC, spend_usd DESC
